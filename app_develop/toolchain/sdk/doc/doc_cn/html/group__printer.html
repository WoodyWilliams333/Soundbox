◆ printer_close()

int printer_close ( int  fd)

关闭打印机设备

参数
fd- 打开设备的句柄 printer_open 函数返回值
返回
int 0 关闭成功 其他 失败
注解
调用此函数后fd将无效，在再次打开打印机设备之前，不能使用此无效fd来操作打印机。

◆ printer_feed_start()

int printer_feed_start ( int  fd,
uint32_t  step 
)

启动打印机送纸功能

参数
fd- 打开设备的句柄 printer_open 函数返回值
step- 送纸步骤
返回
int 
         0       设置成功
         其他       失败
注解
调用此函数后，如果打印机处于空闲状态，则打印机开始进料，打印机内部有一个最大进料步长限制，如果设置的步长大于该步长，则它将根据内部最大步长进料。 进料时，如果开发人员调用printer_feed_stop，打印机将停止进料。

◆ printer_feed_stop()

int printer_feed_stop ( int  fd)

停止打印机进料

参数
fd- 打开设备的句柄 printer_open 函数返回值
返回
int 
         0       操作成功
         其他   失败
注解
如果应用程序开发人员已经调用了printer_feed_start，那么调用该函数将停止打印机进料;如果打印机已停止送料，调用此函数将不对打印机做任何工作。

◆ printer_get_param()

int printer_get_param ( int  fd,
printer_param_t *  param 
)

获取当前打印参数

参数
fd- 打开设备的句柄 printer_open 函数返回值
param-
  1. printer_param_t.ro_width 打印机每一行点阵的点号
  2. printer_param_t.ro_bits_per_pixel 每个像素表示多少位
  3. printer_param_t.rw_gray 设置打印灰，范围:50~500，表示打印灰正常为50%~500%
  4. printer_param_t.rw_hi_temp 打印头允许的最高工作温度
  5. printer_param_t.rw_lo_temp 过温打印后再次打印时，打印头允许的最高温度
返回
int 
         0      获取状态成功，返回参数可用
         其他   失败
注解
When printing , if print head up to the value that rw_hi_temp setted it will be status of "over-temperature", print will stop, until temperature reduce to under rw_lo_temp it will continue to print. If over-temperature occurs , data that have not been printed will all be discarded.

◆ printer_get_status()

int printer_get_status ( int  fd,
printer_status_t *  status 
)

获取打印机状态

参数
fd- 打开设备的句柄 printer_open 函数返回值
status-
  1. printer_status_t.status 状态由32位表示:
    PRINTER_STATUS_BUSY bit：打印设备忙标志.
    PRINTER_STATUS_HIGHT_TEMP bit：温度过高
    PRINTER_STATUS_PAPER_LACK bit：缺纸标志
    PRINTER_STATUS_FEED bit：FEED标志
    PRINTER_STATUS_PRINT bit：印刷标志
    PRINTER_STATUS_FORCE_FEED bit：FORCE_FEED 标志
    PRINTER_STATUS_POWER_ON bit：打印电源开启标志
  2. printer_status_t.voltage 打印电流的电压
  3. printer_status_t.temperature 当前打印头温度
返回
int 
         0       获取状态成功，返回的参数可用
         其他      失败
注解
打印机的获取状态是按位定义的，你应该测试 status->status &（1<<PRINTER_STATUS_BUSY）来确定打印机是否空闲

◆ printer_open()

int printer_open ( const char *  filename,
int  oflags 
)

打开打印机设备

参数
filename- 打印设备名："/dev/printer0","/dev/printer1"……
oflags- 打开设备的方式: 
         只读：  O_WRONLY
         可读可写：O_WRONLY | O_NONBLOCK
返回
int 
         >=0    打开设备OK，返回值用来操作打印机的文件号
         其他   失败
注解
打印机设备的数量最多为16，其设备名为 /dev/printer?（?=0,1,2,…,15），默认情况下NEW8210热敏打印机设备名称为/dev/printer0; 当打印机工作完毕时，必须关闭打印机设备，否则会由硬件关闭。

◆ printer_write()

int printer_write ( int  fd,
const void *  buf,
size_t  nbytes 
)

把要打印的数据传入Printer设备

参数
fd- 打开设备的句柄 printer_open 函数返回值
buf- 指向要打印的数据缓冲区
nbytes- 打印数据的字节数
返回
int 
         >=0     打印机设备实际接受的直接数量
         其他   失败
注解
调用此函数后，打印机将自动开始打印。
在阻塞模式下，该功能可能会阻塞，直到打印机可以接受数据， 在没有例外情况下，打印机将完成打印提交的数据， 如出现异常(如打印出纸、打印头温度过高)， 函数将立即返回。
在非阻塞模式下，打印机将尝试接受要打印的数据， 如果打印机内部打印缓冲区已满，该函数将立即返回已接收的类型的数量， 在内部打印缓冲区已满的情况下，继续调用此函数将返回错误(不是由打印机故障引起的)。
注意，在调用此函数时，打印机可能无法一次性接受所有数据，因此应用程序应该循环调用printer_write，直到打印出所有数据。
