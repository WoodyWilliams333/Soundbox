Returns

◆ printer_get_param()

int printer_get_param ( int  fd,
printer_param_t *  param 
)

get the current print parameters

Parameters
fd- opened device file number that printer_open function returns
param-
  1. printer_param_t.ro_width the dot number of each line dot matrix of printer
  2. printer_param_t.ro_bits_per_pixel how many bits to expressed each pixel
  3. printer_param_t.rw_gray Set print gray, scope: 50~500, represent that print gray is normal 50～500%
  4. printer_param_t.rw_hi_temp highest work temperature that print head allowed
  5. printer_param_t.rw_lo_temp highest temperature that print head allowed when printing again after over-temperature printing
Returns
int 
         0       get status successfully, the returned parameter is available
         other   other error code
Note
When printing , if print head up to the value that rw_hi_temp setted it will be status of "over-temperature", print will stop, until temperature reduce to under rw_lo_temp it will continue to print. If over-temperature occurs , data that have not been printed will all be discarded.

int 
         0       get status successfully, the returned parameter is available
         other   other error code

The got status of printer is defined by bit, you should test status->status &（1<<PRINTER_STATUS_BUSY）to determine wether the printer is ldle

◆ printer_write()

int printer_write ( int  fd,
const void *  buf,
size_t  nbytes 
)

put the data to be printed

Parameters
fd- opened device file number that printer_open function returns
buf- point to data buffer to be printed
nbytes- the number of printed data's bytes
Returns
int 
         >=0     the direct number of printer device actually accept
         other   other error code
Note
After calling this function, the printer will automatically begin to print.
In the blocking mode, the function may block until the printer can accept data, on no exceptional condition, printer will complete print submitted data, if an exception occurs(such as printing out of paper,print head temperature is too high), function will return immediately.
In the non-blocking mode, the printer will try to accept the data to be printed, if the printer internal print buffer is full, this function will immediately return the number of types have been received, in the case of internal print buffer is full, continue to call this function will return error(not by the failure of the printer ).
Note that calling this function, the printer may not accept all of the data one time, so application should loop call printer_write, until all of data are printed.
