◆ bt_user_confirm_passkey()

int bt_user_confirm_passkey ( int  fd,
void *  data,
int  len 
)

非阻塞获取GKEY

Parameters
[in]fd设备的文件描述符
[in/out]data : 存储GKEY的buffer
[in]len: buffer大小，不能小于GKEY长度4字节
Returns
返回值
>=0 获取GKEY的实际长度
else -—failed, 可根据errno获取错误码
Note
: 如果返回值为-EAGAIN(-11),表示需要进一步尝试获取
CONFIRM GKEY配对方式为
1.先调用bt_pairing_mode_set() 设置配对模式为PAIRING_MODE_CONFIRM
2.再反复调用bt_user_confirm_passkey() 获取GKEY
3.手机端发起配对请求后，bt_user_confirm_passkey() 获取GKEY成功
4.对比手机端显示的GKEY与获取的GKEY, 调用bt_user_confirm() 确认获取的GKEY是否正确
5.确认GKEY无误则配对成功，否则失败
