◆ BarcodeMode

enum BarcodeMode

This enumeration lists the modes supported by the scan interface.


Enumerator
BAR_MODE_CONTINUE 

Support one-dimensional code, two-dimensional code, continuous scanning code

BAR_MODE_ONCE 

Support one-dimensional code, two-dimensional code, only scan once after open

BAR_MODE_MOTIONDETECT 

Not supported. One-dimensional code and two-dimensional code are supported. Scan codes are emitted when image brightness changes are detected, otherwise the same as BAR_MODE_CONTINUE

BAR_MODE_ONED_ONLY 

BAR_MODE_CONTINUE is not supported. Only one dimension code is scanned, otherwise the same as BAR_MODE_CONTINUE

BAR_MODE_MOTIONDETECT_MINI 

This parameter is not supported. It is the same as BAR_MODE_MOTIONDETECT and is not recommended

Setting Bar Code Parameters.


Parameters
[in]fd- the descriptor
[in]cmd- BAR_IOCTL_SET_SOUND - Set scan sound(arg:1 sound, arg:0 silent);
BAR_IOCTL_SET_READ_CALLBACK - Sets the callback function after a successful scan，In the form of bar_callback_t
[in]arg- With CMD corresponding
Returns
0 - success
-1 - fail
int fd;
int res;
fd = bar_open("/dev/barcode", BAR_MODE_ONCE); //Scan only once after open
res = bar_ioctl(fd, BAR_IOCTL_SET_PREVIEW, TRUE); //TRUE:Open the preview FALSE:Close the preview
if (!res) {
//---- display code ----
lcd_flip();
}
bar_close(fd);
